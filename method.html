本站致力于建立一个公正、公开、透明的机场可靠性排名平台。所有排名数据均来自部署于不同云厂商的多节点自动化监测脚本，每隔 30 分钟全自动运行一次，对每个机场的官网与订阅链接进行探测。为评估一个机场的「可靠性分值」，我们采用 6 个维度的加权计算，并对新入驻机场的初始评分引入确定性锚点，避免所有新机场千篇一律得同一分数。

⏱

全天候可用性监测

每 30 分钟，监测脚本从美国、新加坡、日本、中国香港等 4 个机房节点，向机场官网及订阅地址发起探测。

📡

订阅可用率（40分）

评估订阅内容获取是否顺畅。如果订阅解析接口频繁超时或返回空配置，将会导致分值严重下滑。Clash/V2Ray 格式均覆盖。

🌐

官网可用率（25分）

官网是用户充值、找回密码、更换配置的唯一通道。官网的可用性与是否被污染直接影响最终评分，大面积DNS污染将触发扣分。

📅

运营稳定性（12分）

老牌机场拥有更丰富的运营经验和冗余带宽。新机场在初期会获得新秀分值保护，但高风险机场将被直接扣减分值。

可靠性评分区间说明

SCORE RANGE INTERPRETATION

90 ~ 92 分
⭐ 卓越稳定（极难达到）

需要持续高可用率（>98%）+ 长期稳定运营（180天+）+ 低延迟专线，极少数成熟优质机场才能达到此区间。

85 ~ 90 分
✅ 优质推荐

官网与订阅可用率稳定，响应速度良好，整体表现出色。大多数长期可靠机场处于此区间，适合日常重度使用。

75 ~ 85 分
🔶 正常可用

新入驻机场或某一维度存在短板（如响应较慢、价格偏高、运营时间尚短），可正常使用，建议持续关注其稳定性变化。

75 分以下
❌ 谨慎使用

多维度表现欠佳，或存在风险预警扣分（-50分），建议密切关注其状态后再决定是否购买。

自动监测与评分更新流程

AUTOMATED PIPELINE · EVERY 30 MINUTES

1

触发监测（每30分钟）

GitHub Actions 定时任务触发 check.py 脚本，多地云服务器并发启动探测任务，最大并发10个机场同时检测。

2

双通道探测

对每个机场同时执行：① 官网 HTTP 可达性测试（记录状态码+延迟ms）；② 订阅链接内容拉取测试（验证返回内容有效性）。

3

写入数据库

探测结果写入 Supabase speed_logs 表，每条记录含：airport_id、检测时间、website_ok、website_ms、sub_ok、HTTP 状态码。

4

六维评分计算

读取该机场近30天 speed_logs，结合其价格、标签、运营天数，代入6维算法计算新得分，并与前次对比生成 score_delta（↑↓变化量）。

5

更新排名展示

新评分写回 airports 表，前端通过 Supabase API 实时拉取最新分数并按分值降序排列，用户刷新页面即看到最新榜单。

常见问题

FAQ

付费能提高排名吗？

不能。排名完全由自动化监测数据决定，本站商业模式为 CPC 点击广告（机场出价竞争首位广告展示位），与可靠性评分完全隔离。评分高低只取决于机场的实际表现数据。

新入驻机场初始分是多少？

新机场无监测数据时，订阅可用率取 88%~98% 之间的确定性哈希初始值，官网可用率取 92%~100%，响应速度取 6~12 分，所有值均由机场 ID 唯一确定，不同机场不会完全相同，避免全部堆叠在同一分数。

评分为什么突然大幅下降？

最常见原因：① 官网或订阅在过去30天内出现大量失败记录；② 被识别为风险机场，触发 -50 分惩罚；③ 价格调整后价格竞争力分降低。您可在机场报告页查看详细的历史可用率曲线。

我发现某机场数据有误，怎么反馈？

请发送邮件至 admin@jctuijian.com 并附上机场名称和问题描述，我们会在24小时内核实并处理。

风险预警是怎么触发的？

当出现以下任一信号时自动触发：官网连续 3 天完全不可达、订阅链接返回空内容超过72小时、TG 官方群解散、用户集中投诉且无官方回应超过48小时。触发后扣 50 分并在首页置顶风险提示卡。
